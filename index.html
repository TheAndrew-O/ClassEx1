Part 2 - 2D Single Agent Navigation

In order to navigate the agent along the path while not colliding with obstacles, the TTC algorithm was used.
The agent is provided a list of targets to move towards and a list of obstacles to avoid.
When the agent reaches a target/goal, the next node in the list of goals is set as the goal for the agent to move towards.
I had trouble making the agent acceleration not stop and start at each node.
I attempted to smooth the transitions between nodes by modifying the goal force attracting the agent.

3D Simulation of A* Pathfinding in Unity

The following features were implemented

3D Rendering & Camera

The lighting is cast at an angle to simulate natural lighting and shadows.
There are 4 cameras, they are mapped to the numbers 1-4. 2 cameras show different angles of the map. 2 cameras follow a tank at different angles.

Improved Agent & Scene Rendering

There are three agents, 2 of them are 3D low-poly tanks and the other is a capsule.
The obstacles are 3D low-poly rocks and buildings. I did not make these assets, they are from the Unity asset store.
The assets came with mesh colliders, but the obstacle avoidance was done mostly by me.
The obstacle avoidance was implemented by creating an unwalkable layer-mask and applying it to the grid coordinates of where the obstacles are located.
I then used Unity's built in Physics library to call the CheckSphere function which will check if a node is walkable, given an unwalkable layer-mask.

Incorporate Particle Systems

I tried to implement particles in Unity from scratch. It was difficult and I was only getting particles with missing textures.
I instead used the built in particle system to spawn particles in the smokestack.

Orientation Smoothing

To orientate the tanks towards their goals, the angle of rotation was calculated using the arctangent.
To smooth the orientation so that it didn't immediately snap to the new direction, I used the function SmoothDampAngle
with Euler's function to calculate the angles of rotation needed.

Multiple Agents Planning

To process multiple agents moving simultaneously, a movement handler was created to process each path created.
I ran into major issues with the game stalling for a second or two as it tried to calculate the paths.
To fix this issue a handler was implemented to to process each path at seperate frames, not all at once on the same frame.
I then have another class to handle the tank's movement.
